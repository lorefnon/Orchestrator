Orchestrator

An extension for Backbone View class to provide enhanced facilities for methodically managing the DOM structure and Data associations between Model and View.

View project onGitHub

Overview :

Orchestrator aims to be a simple extension for Backbone View class which enables you bind model attributes to content, attributes or css-rules for any DOM node. Binding configuration is specified in pure javascript inside View class. No custom attributes / special tags / exotic markup is required in the HTML.

Orchestrator does not seek to perform any kind of occult sorcery behind the scenes. While this does mean that everything is pretty much transparent and for cases of advanced configuration debugging is easy, it also means that no claim regarding drastic reduction of code is being made.

Orchestrator is agonistic of any templating libraries and has built in support for asynchronous construction of DOM nodes. While basic use-cases can be fulfilled in very few lines of code, the configuration system is flexible and can accomodate a wide variety of advanced usage scenarios. Asynchronous construction of DOM nodes is supported out of the box, so you can pull in your templates when necessary, create the relevant nodes and once done bind them to the model. Also real-time binding is supported for elements which might be added on-the-fly.

Alternatives :

There are other projects which have been tackling the problem of data-binding from a variety of aspects.

If your prefer to specify your binding configuration through data-attributes right inside your HTML - Knockout JS is worth a look. An integration layer for using Knockout alongside Backbone is available. Another similar 5approach that is easy to integrate in any MVC setup is offered by Rivets.js library.

AngularJS by Google takes a much more radical and sophisticated approach towards building client side web-applications. If you do like the concept of supercharging HTML with capabilities beyond its original intent, this is definitely worth a try.

Contributions :

You are welcome to contribute to the project if you like the concept. Fork the project and once you have made your changes, request for a pull. Please make sure that your changes/contributions available under the MIT license. It is preferred that your contributions would not introduce large dependencies. I plan to keep the codebase tiny, lightweight and maintainable. Also for the sake of coherence please follow the same coding style as in Backbone source code.

Also apart from code contributions - comments, suggestions and bug-reports are welcome too. Please use the Github issue tracker for the same.
